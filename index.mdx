## What is sheets-mcp?

**sheets-mcp** is a hosted MCP (Model Context Protocol) server that gives any AI agent — Claude, Cursor, or your own — complete programmatic control over Google Sheets.

It exposes **15 battle-tested tools** that cover 100 % of the Sheets API v4 surface without flooding the agent context window. Two lines in your MCP config and your agent can read, write, analyze, and structurally transform spreadsheets using natural language.

## Why sheets-mcp?

<AccordionGroup>
  <Accordion title="Context-window safe by design">
    Every tool declares its output schema (June 2025 MCP spec). The `analyze_range` and `transform_range` tools run SQL directly against Sheets data in Postgres — your agent never has to pull 50k rows into its context.
  </Accordion>
  <Accordion title="Fails closed, not open">
    `write_range` and `transform_range` block writes that would silently destroy formula cells or spill into ARRAYFORMULA ranges. You opt in to overrides explicitly with `allowFormulaOverwrite: true`.
  </Accordion>
  <Accordion title="Two auth paths">
    Use OIDC (OAuth2 + JWT) for interactive clients like Claude Desktop and Cursor. Use an API key (`x-api-key` header) for headless scripts, CI pipelines, or custom agents.
  </Accordion>
  <Accordion title="Batch-first API surface">
    Tools prefer batch operations over sequential calls — Sheets API charges per HTTP request. `batch_update_sheet` exposes the full `batchUpdate` surface for charts, merges, and styling in a single round-trip.
  </Accordion>
</AccordionGroup>

## Stack

Built on **Next.js 16** deployed to Vercel. Auth via **better-auth** with Google OAuth. Data pooling via **Supabase Postgres**. Observability via **OpenTelemetry**.
