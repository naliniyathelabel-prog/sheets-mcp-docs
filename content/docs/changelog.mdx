## [1.1.0] — 2026-04-16

### Added

### append_rows — non-destructive row insertion

Appends 2D string arrays into existing ranges using `insertDataOption: INSERT_ROWS`. Unlike `write_range`, it never overwrites existing rows — it finds the bottom of the data table and inserts below it.

    **AX impact:** Agents can now safely add rows to a live table without knowing the current row count. No need to `read_range` first to find the last row.

### clear_range — value wipe, formatting preserved

Clears cell values in a bounding box while explicitly preserving all formatting, data validation rules, and conditional formatting.

    **AX impact:** Agents can reset data in a template sheet without destroying the visual structure the user has set up.

### copy_sheet — tab duplication

Duplicates an existing worksheet tab into a new spreadsheet using `spreadsheets.sheets.copyTo`. Returns the new sheet's ID.

    **AX impact:** Enables template workflows — copy a formatted report tab, then populate it with `write_range`.

### batch_update_sheet — full batchUpdate passthrough

Raw proxy for the Google Sheets `batchUpdate` endpoint. Accepts an array of any `batchUpdate` request objects — charts, merges, dimension resize, conditional formatting, data validation, and more.

    **AX impact:** Exposes the full structural API surface that was previously inaccessible through single-purpose tools. Agents can now build charts and apply rich formatting in a single round-trip.

### analyze_range — context pagination protection (SQL engine)

The primary defence against LLM token context exhaustion. Fetches raw sheet data via Sheets REST API, loads it into a Postgres JSONB CTE (`sheet_data`), executes native SQL aggregations, and returns only the result to the agent context.

    The fetch result is cached in Supabase (`sheet_snapshots`) for **60 seconds** — repeated calls within the window skip the Sheets API entirely.

    **AX impact:** Agents can now summarise, aggregate, and filter datasets of any size without a single raw row entering the context window.

### Changed

- **Schema token optimisation (41% reduction):** Refactored all Zod schema payloads in `route.ts`. Trimmed redundant `z.describe()` attributes on self-evident fields. Total schema context load dropped from ~760 tokens to ~450 tokens per tool invocation. This directly increases the amount of reasoning budget available to the agent per call.

- **Vercel build stability:** Fixed TypeScript strict-mode type errors in schema payload mapping that caused intermittent build failures on Vercel Edge.

### Security

- **`analyze_range` SQL injection guard:** Queries are sanitised before execution — `;` is stripped (blocks statement chaining), and `DROP`/`DELETE`/`UNION` are rejected via regex. The stripping step was carefully designed not to break column names containing the word "dropoff" or similar legitimate values.

- **Read-only Postgres transaction:** All `analyze_range` queries run inside `BEGIN READ ONLY` + `SET LOCAL ROLE sheet_analyzer`. Even if the SQL injection guard is bypassed, Postgres enforces the read-only constraint at the transaction level.

- **Context safety clamp:** `LIMIT 1000` is auto-appended to every `analyze_range` query, preventing runaway result sets from consuming agent context.

- **Vercel deploy hygiene:** Purged legacy `dev@example.com` commit identity from git history via interactive rebase to unblock Vercel Edge deployment security checks.

## [1.0.0] — 2026-04-13

Initial production release.

### Features

- **`list_spreadsheets`** — List all spreadsheets in the authenticated user's Google Drive
- **`list_sheets`** — List tabs inside a spreadsheet
- **`read_range`** — Read a 2D cell range with `FORMATTED_VALUE`, `UNFORMATTED_VALUE`, or `FORMULA` render modes
- **`write_range`** — Write a 2D array to a specific range with formula overwrite protection
- **`create_spreadsheet`** — Create a new blank spreadsheet
- **`format_cells`** — Apply visual formatting via zero-based grid indexes
- **`manage_sheets`** — Add, rename, or delete tabs in a single tool call
- **`manage_drive`** — Search Drive files and convert non-native files to Sheets
- **`restore_snapshot`** — List and restore pre-write snapshots from `write_history`
- **`transform_range`** — Write-enabled SQL engine with pre-write snapshot and formula protection

### Architecture

- Next.js 16 deployed on Vercel (single-file 787-line MCP transport)
- `better-auth` as OIDC provider + API key plugin
- Supabase Postgres for session storage, snapshot cache, and write history
- `googleapis` JS SDK with proactive token refresh on every invocation
- OpenTelemetry instrumentation (OTLP HTTP exporter)
- Two auth paths: OIDC (Claude Desktop / Cursor) + `x-api-key` header

### Database migrations

- `2026-04-13T03-22-30.752Z.sql` — Initial schema (sessions, accounts, write_history)
- `2026-04-13T06-41-17.610Z.sql` — `sheet_snapshots` table with TTL and GIN index on JSONB data
