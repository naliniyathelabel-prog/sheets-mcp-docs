## Prerequisites

- [Cursor](https://cursor.sh) IDE installed (v0.43 or later recommended)
- A sheets-mcp account at [sheets-mcp-xi.vercel.app](https://sheets-mcp-xi.vercel.app)

---

<Steps>

  <Step title="Open Cursor MCP settings">

    Go to **Cursor → Settings → Features → MCP Servers**.

    Or open the Command Palette (`Cmd+Shift+P` / `Ctrl+Shift+P`) and search **MCP: Add Server**.

  </Step>

  <Step title="Add a new MCP server">

    Click **Add new MCP server** and enter:

    - **Name:** `sheets-mcp`
    - **URL:** `https://sheets-mcp-xi.vercel.app/api/mcp`
    - **Transport:** Auto (Cursor negotiates SSE or HTTP automatically)

    Or edit `~/.cursor/mcp.json` directly:

    ```json
    {
      "mcpServers": {
        "sheets-mcp": {
          "url": "https://sheets-mcp-xi.vercel.app/api/mcp"
        }
      }
    }
    ```

  </Step>

  <Step title="Reload Cursor">

    `Cmd+Shift+P` → **Developer: Reload Window** to apply the config.

  </Step>

  <Step title="Authorize via Google OAuth">

    The first time Cursor calls a sheets-mcp tool, it opens the OIDC authorize URL in your browser. Sign in and grant Sheets + Drive scopes.

    Once authorized, the session is stored in Supabase and reused on all subsequent calls — no repeated consent.

  </Step>

  <Step title="Test in Cursor Chat">

    Open a Cursor Chat window (`Cmd+L`) and type:

    > "Use sheets-mcp to list my Google spreadsheets"

    Cursor's agent calls `list_spreadsheets` and displays results inline.

  </Step>

</Steps>

## Using an API key (headless / CI)

Generate an API key from [sheets-mcp-xi.vercel.app/dashboard](https://sheets-mcp-xi.vercel.app/dashboard).

The key is shown **once** — copy it immediately.

```json
{
  "mcpServers": {
    "sheets-mcp": {
      "url": "https://sheets-mcp-xi.vercel.app/api/mcp",
      "headers": {
        "x-api-key": "YOUR_API_KEY"
      }
    }
  }
}
```

API keys have no expiry by default and resolve to the same Google OAuth session as OIDC. See [Authentication](/authentication) for full details.

## Troubleshooting

| Symptom | Fix |
|---|---|
| Server shows as "disconnected" | Check the URL has no trailing slash. Reload the Cursor window. |
| OAuth window does not open | Cursor uses the system browser. Ensure your default browser is set. |
| Tools visible but returning errors | Open Cursor DevTools (`Help → Toggle Developer Tools`) and check the MCP panel. |
| `needsAuth: true` in response | Refresh token revoked. Sign out from dashboard and re-authorize. |
