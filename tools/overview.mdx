## Design principles

sheets-mcp's tool surface was designed against three constraints:

1. **Context window budget** — Every tool schema token costs inference budget. The full schema payload was optimised from ~760 tokens down to ~450 tokens (41% reduction) by trimming redundant descriptions and `z.describe` attributes on self-evident fields.
2. **Sheets API cost** — The Sheets API charges per HTTP request. Tools prefer batch operations (`batch_update_sheet`, `append_rows` with `INSERT_ROWS`) over sequential single-cell writes.
3. **Agent ergonomics** — Tool names and parameter names follow the principle of least surprise. Overlapping operations are merged (e.g. `delete_sheet` + `add_sheet` + `rename_sheet` → `manage_sheets`).

### 2026 MCP spec features used

| Feature | How sheets-mcp uses it |
|---|---|
| **Tool Output Schemas** (June 2025 spec) | Every tool declares its return shape to reduce context waste |
| **Structured errors** | All errors return `{ code, message, details }` — never raw stack traces |
| **Batch-first** | `batch_update_sheet` covers the full `batchUpdate` surface in one round-trip |
| **Data-in-place operations** | `analyze_range` and `transform_range` run SQL inside Postgres — sheet data never enters the agent context |

## Rate limits

The underlying Google Sheets API enforces **60 read/write requests per minute per user**. sheets-mcp does not add its own rate limit layer on top of this (scheduled for a future release). If you exceed the Google quota, tools return a structured `429` error with a retry hint.

## Error format

All tool errors follow a consistent shape:

```json
{
  "content": [
    {
      "type": "text",
      "text": "{ \"code\": \"PERMISSION_DENIED\", \"message\": \"The caller does not have permission\", \"details\": {} }"
    }
  ],
  "isError": true
}
```

See the [Error Reference](/reference/errors) for the full list of error codes.
