sheets-mcp is built on a **fail-closed** principle: when a write operation is ambiguous or potentially destructive, the tool blocks the operation and returns a structured error instead of proceeding silently.

---

## Formula overwrite protection

### What it prevents

Silently overwriting a cell that contains a formula (e.g. `=SUM(A1:A10)`) with a static value is one of the most common ways AI agents corrupt spreadsheets. Once a formula is replaced, the original cannot be recovered without version history.

### How it works

Before every `write_range` or `transform_range` call, sheets-mcp:

1. Fetches the target range using `valueRenderOption: FORMULA`
2. Scans every cell for a leading `=` character
3. If any formula cells are found, the write is **blocked** with this error:

```
write_range blocked because the target range contains formula cells (3):
B2, C2, D2. Pass allowFormulaOverwrite: true only if you intend to
replace formulas with static values.
```

### Bypassing safely

If you genuinely want to replace formulas with static values, pass `allowFormulaOverwrite: true` explicitly:

```json
{
  "tool": "write_range",
  "spreadsheetId": "...",
  "range": "Sheet1!B2:D2",
  "values": [["100", "200", "300"]],
  "allowFormulaOverwrite": true
}
```

<Warning>
  This bypass is permanent. Once a formula is overwritten with a static value, it cannot be restored without Google Sheets version history.
</Warning>

## ARRAYFORMULA spill protection

### What it prevents

`ARRAYFORMULA` cells automatically expand their output into adjacent cells. Writing static data into those downstream cells silently breaks the formula's output range.

### How it works

sheets-mcp detects ARRAYFORMULA patterns in the sheet schema (cached for 300 seconds). If your write target overlaps a column that contains an ARRAYFORMULA in row 2, the write is blocked:

```
⚠️ A target column contains an ARRAYFORMULA. Writing static values will
permanently destroy this formula. Pass allowFormulaOverwrite: true to
confirm you intend to replace the formula with static data.
```

The schema cache stores `arrayFormulaColumns` (e.g. `["B", "C"]`) so this check runs without an extra API call on repeated writes.

## Protected Range enforcement

Google Sheets allows users to lock ranges so only specific editors can modify them. sheets-mcp respects these locks:

| Protection type | Behaviour |
|---|---|
| **Locked (strict)** | Write is **blocked**. Error: `"range overlaps a Protected Range (locked). The sheet owner has locked this range. This cannot be overridden."` |
| **Warning-only** | Write proceeds but returns `⚠️ Target range has a user-set protection warning.` |

Protected range data is fetched via the schema cache and checked before every write.

## SQL injection guard (`analyze_range` / `transform_range`)

`analyze_range` lets agents run arbitrary Postgres SQL against sheet data. To prevent abuse:

1. **Statement chaining blocked** — the `;` character is stripped from all queries before execution
2. **Mutation keywords blocked** — `DROP`, `DELETE`, `UNION` are rejected via regex before the query reaches Postgres
3. **Read-only transaction** — all queries run inside `BEGIN READ ONLY` — even if a mutation somehow bypassed the regex, Postgres would reject it
4. **Hard row limit** — every query has an automatic `LIMIT 1000` appended, preventing runaway result sets from overwhelming the agent context

```sql
-- This query will be blocked before execution
DROP TABLE sheet_data; SELECT * FROM pg_user;

-- This query runs safely
SELECT data->'Category' AS cat, COUNT(*) AS total
FROM sheet_data
GROUP BY cat
ORDER BY total DESC
LIMIT 20
```

## Response size clamp

Every tool that returns row data runs through `truncateIfNeeded`:

- Hard limit: **1,000 rows** or **2 MB**, whichever is hit first
- If truncated, the response includes a warning:

```
⚠️ read_range: Truncated to 1000 rows (original: 4821).
Use a narrower A1 range or analyze_range with SQL for large datasets.
```

This prevents a single large `read_range` call from consuming the entire agent context window.
